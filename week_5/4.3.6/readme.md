# Модуль билетов (`tickets.py`)

Данный модуль реализует иерархию классов для моделирования различных типов проездных билетов. Включает базовый абстрактный класс и конкретные реализации с разными ограничениями.

---

## Классы

### 1. `ПроезднойБилет` (абстрактный класс)
- **Описание:** Базовый класс для всех типов билетов.
- **Поля:**
  - `цена` (публичное): стоимость билета.
  - `__использовано_поездок` (приватное): количество использованных поездок.
  - `_ограничение_поездок` (защищенное): лимит поездок, может быть `None`.
- **Методы:**
  - `списать_поездку()`: абстрактный метод, реализуемый в подклассах.
  - `показать_статус()`: выводит текущий статус билета.
  - `_увеличить_использование()`: защищенный метод для увеличения количества использованных поездок.

---

### 2. `БилетСОграничением`
- **Описание:** Реализация билета с ограничением по количеству поездок.
- **Наследует:** `ПроезднойБилет`
- **Поля:**
  - `_ограничение_поездок`: лимит поездок.
- **Методы:**
  - `списать_поездку()`: списывает поездку, если лимит не достигнут.
  - `_get_used()`: возвращает количество уже использованных поездок.

---

### 3. `БилетСОграничениемПоездок`
- **Описание:** Конкретная реализация билета с ограничением по количеству поездок.
- **Наследует:** `БилетСОграничением`
- **Особенности:** 
  - При вызове `списать_поездку()` проверяет, не достигнут ли лимит, и списывает поездку.

---

### 4. `БезлимитныйБилет`
- **Описание:** Билет без ограничений по количеству поездок.
- **Наследует:** `ПроезднойБилет`
- **Особенности:**
  - Метод `списать_поездку()` всегда успешно списывает поездку без ограничений.

---

## Использование
Создавайте экземпляры классов и вызывайте их методы для моделирования поведения билетов:
```python
ticket = БилетСОграничениемПоездок(100, 5)
ticket.показать_статус()
ticket.списать_поездку()
