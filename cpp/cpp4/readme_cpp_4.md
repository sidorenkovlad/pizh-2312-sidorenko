# Расширенная реализация телефонного справочника с полиморфизмом

Данная программа реализует телефонный справочник с использованием принципов объектно-ориентированного программирования, включая наследование, полиморфизм и динамическое управление памятью.

---

## Описание классов

### Базовый класс `Person`

- **Назначение:** Абстрактный базовый класс, определяющий интерфейс для всех типов объектов, которые могут храниться в справочнике.
- **Данные:** отсутствуют.
- **Методы:**
  - `virtual ~Person() {}` — виртуальный деструктор для правильного освобождения памяти при удалении объектов через указатели на базовый класс.
  - `virtual void print() const = 0;` — чисто виртуальная функция, которая должна быть переопределена в наследниках для вывода информации о объекте.

### Класс `Kontakt` (из лабораторной №3)

- **Назначение:** Представляет контакт с информацией о человеке.
- **Данные:**
  - `name` — имя контакта.
  - `phone` — номер телефона.
  - `address` — адрес.
- **Конструкторы:**
  - По умолчанию, с параметрами, копирующий.
- **Методы:**
  - `input()` — ввод данных с клавиатуры.
  - `display()` — вывод данных на экран.
  - `print()` — переопределенная функция из базового класса, вызывающая `display()`.

### Класс `Employee`

- **Назначение:** Представляет сотрудника организации.
- **Данные:**
  - `name` — имя сотрудника.
  - `salary` — зарплата.
- **Методы:**
  - Конструкторы по умолчанию и с параметрами.
  - `print()` — выводит информацию о сотруднике.

### Класс `Customer`

- **Назначение:** Представляет клиента организации.
- **Данные:**
  - `name` — имя клиента.
  - `orderNumber` — номер заказа или идентификатор клиента.
- **Методы:**
  - Конструкторы по умолчанию и с параметрами.
  - `print()` — выводит информацию о клиенте.

---

## Использование

В основной функции программы создается массив указателей на базовый класс `Person`. В него помещаются объекты всех типов наследников (`Kontakt`, `Employee`, `Customer`). В цикле вызывается виртуальная функция `print()`, что обеспечивает полиморфизм и правильный вызов соответствующих методов для каждого объекта.

Также реализован класс `Spravochnik`, который содержит вектор указателей на объекты типа `Person`. Он позволяет добавлять любые объекты-наследники и выводить всю информацию о них через единый интерфейс.
